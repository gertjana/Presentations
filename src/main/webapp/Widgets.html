Widgets

Widgets are Component Presentations that have (or are):

  • a defined interface to the outside world.
  • able to connect to a Data Source.
  • able to consume a generic Data model.
  • manageable as a single entity.
  • of a static or dynamic nature.

Static Widgets

... are rendered by the Publisher, the functionality is in your (modular) Templates

Examples

  • A list of ComponentPresentations
    Rendered in your template
  • Static content
    don't really need a widget for this
  • Content from a external system.
    Stock counts from your inventory system

Dynamic Widgets

... are rendered runtime, the functionality is in your jsp, aspx, or rel

Examples

  • A list of ComponentPresentations
    but now by doing a broker query
  • Ambient Data Framework: Audience Management User profile
    reads the AM Data from the ADF at runtime
  • Webservice: Weather Widget
    connects to a external webservice

Modular Templates



A 'Widget Finish Actions' Template building block will render static widgets
Otherwise it will leave <tcdl:...> tags in.

From Template to webpage

Widget Configuration (Embedded schema)
Name User Profile
DataSource adf://taf:claim:audiencemanager

Here the Data source is the Ambient Data Framework
more specific all claims belonging to audience manager

Another examples of data sources
http://weather.com/api/localweather.xml
component://Component.Fields.FieldName

From Template to webpage 2

Dreamweaver 
@@RenderWidget()@@
    <div class="userprofile">
        @RenderWidgetField("contact:firstname", "text")&nbsp;
        @RenderWidgetField("contact:lastname", "text")<br>
        @RenderWidgetField("contact:email", "email-link")
    </div>
@@EndRenderWidget()@@
A Template building block will take care of the transformation to tcdl tags
(In case of a static widget just render the content)

From Template to webpage 3

(Published) Template 
<tcdl:widget name="user profile" protocol="adf" data="taf:claim:audiencemanager">
    <div class="userprofile">
        <tcdl:widgetField value="contact:firstname" decorator="text" />&nbsp;
        <tcdl:widgetField value="contact:lastname" decorator="text" /><br>
        <tcdl:widgetField value="contact:email" decorator="email-link" />
    </div>
</tcdl:widget>
