Reactive Programming - Resilient


  • Failures are just events and should be handled as such
  • This will require a supervisor strategy
scala 
class Supervisor extends Actor {
  override val supervisorStrategy =
    OneForOneStrategy(maxNrOfRetries = 10, withinTimeRange = 1 minute) {
      case _ : ArithmeticException => Resume
      case _ : NullPointerException => Restart
      case _ : Exception => Escalate
    }
}

Coding in style (Scala)

  • Expressions not statements
  • Try returns a success or failure, you can compose them with for
scala 
  def first() = {
    Try(Console.readline("add a nr:")
  }
  def second() = {
    Try(Console.readline("add a nr:")
  }
  def sum:Try[Int] = {
      for (
        i <- first
        j <- second
      } yield i.toInt + j.toInt
 }
