TomEE - a Java EE6 Application Server based on Tomcat

  • Standard Tomcat with added enterprise features (EJB, JPA, CDI, JSF, WS-*)
  • Passes the (Web profile) EE6 Certification tests (tests run on 100+ amazon EC2 micro instances with arquillian and run with default mem settings with the hour)
  • What's not in it: CMP, CORBA, JAX-RPC
  • What's in the plus package: JAX-RS, JAX-WS, JMS
  • Fast and Lean and still looks and acts like Tomcat
    (27Mb package that runs in 64Mb of memory)
  • Convention over configuration: The minimum configuration needed is: 
    <tomee></tomee>
  • Integration in Maven and Arquillian

Java Enhancements - new Java API's (couple of talks combined)

  • JSR 353: Java API for JSON Parser
  • Stream based, like the StAX Parser
    register to Events to handle objects as they come in
    Probably in JavaEE 7
  • JSR 356: Java API for websocket
  • Simple protocol to send/receive text/binary, some async support works by opening a standard http connection and then upgrade to websocket
    Both Annotation based and interface based API
    will be in JavaEE 7

Code Theory - Functional Compilers from CFG to EXE

  • Talk focused on creating a simple compiler in Scala
  • Compilers typically compile something that we don't know how to handle into something we do know how to handle.
  • 1. Tokenize your source file.
  • 2. Create a AST (Abstract Syntax Tree) from it
  • 2b. If needed transform the tree into another tree.
  • 4. Generate code (not in scope for this meeting as it is the easiest part)
  • The barefooted speaker ended almost every sentence with " and that's fun" especially when talking about complex stuff

Behaviour Driven Development

What if Developer, Tester, Product Owner, Business analyst all speak the same language?

  • Build only features that add value
  • Result in less wasted effort (through traceability)
  • Improve communication
  • Get a higher quality, better tested product

User Story
As a [Role] I want [Feature] So that I receive [Value]

Specification
Given [Context] When [Event Occurs] Then [Outcome]

Behaviour Driven Development 2

java (JBehave)
